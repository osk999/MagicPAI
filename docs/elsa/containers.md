## Containers

Elsa provides multiple deployment options for developers. Beyond integrating its packages into custom .NET applications, the platform offers prebuilt Docker images designed for streamlined deployment.

These Docker images are available in several configurations to meet different use cases:

- **Standalone Elsa Server** — dedicated to API-based workflow execution
- **Elsa Studio** — provides a user-friendly design interface
- **Combined Setup** — integrates both Elsa Server and Studio together

> **Note:** Worker containers used by MagicPAI mount the host repository at `/workspace` inside the container, so all build and test commands run against the live source tree.

According to the source material: "Elsa not only allows for the integration of its packages into custom .NET applications, but also offers prebuilt Docker images for ease of deployment."
